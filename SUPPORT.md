# Support

## Bug Reports

Use the GitHub bug report issue template when behavior is incorrect, flaky, or unsafe.

Include:

- the command you ran
- relevant stdout or stderr
- platform details
- `codex --version` and `codex-switch --version`
- steps to reproduce

## Feature Requests

Use the feature request template for new commands, workflow improvements, or control-surface changes.

## Usage Questions

Open a GitHub issue with context about the workflow you are trying to support. If the question is really a bug or feature request, label it that way.

## Security Issues

For vulnerabilities, follow [SECURITY.md](./SECURITY.md) and avoid public disclosure before a fix is available.
